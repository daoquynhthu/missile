# RoboMaster 飞镖专家系统 (RM Dart Expert)

本 Skill 旨在指导 Agent 快速上手 AeroSim 项目中的 **RoboMaster 飞镖子系统**，确保开发过程符合物理严谨性并遵循已建立的架构规范。

## 1. 核心架构原则

### 1.1 仿真剖面 (Simulation Profile)
- **准则**：必须区分 `GLOBAL_BALLISTIC` (远程导弹) 与 `LOCAL_TACTICAL` (近程飞镖)。
- **物理差异**：飞镖模拟必须禁用科氏力、离心力，使用 **相对 ECEF 积分** 以保证毫米级精度。
- **重力模型**：飞镖采用 NED 系下的局部精确重力，根据发射点经纬度 (如 22.58, 113.96) 通过 `GravityModel` (EGM2008) 计算得出，严禁硬编码 9.8。

### 1.2 坐标系定义
- **NED (North-East-Down)**：局部导航系。X轴北，Y轴东，Z轴向下。
- **体轴系 (Body Frame)**：X轴沿镖身指向头端。
- **姿态表示**：统一使用四元数 $q_{be}$ (Body to ECEF)。
- **初始对齐**：发射瞬间必须确保体轴 X 轴与初速度矢量 $V_{ned}$ 严格重合，以消除初始大迎角导致的数值爆炸。

## 2. 气动模型规范

### 2.1 几何定义
- **鼻端**：采用胶囊形 (Capsule) 模型，长宽比约 3:2 或 4:5。
- **参考面积**：统一使用截面积 $S_{ref} \approx 0.0006 \text{ m}^2$。
- **查找表**：数据位于 `data/dart/rm_dart_aero_table.csv`，由 `scripts/dart/generate_dart_aero_table.py` 生成。

### 2.2 气动力映射
- **阻力 ($C_x$)**：在体轴系中为负值。
- **俯仰力矩 ($C_m$)**：必须具备静态稳定性（迎角增加，力矩减小）。
- **舵效**：抬头指令 (Pitch Up) 对应加速度 $a_z < 0$ (NED) 和负的俯仰力矩增量。

## 3. 调优与验证流程

### 3.1 严谨验证步骤
1. **单轨迹审计**：使用 `RMDartDebug` 生成逐行日志，对比 $t=0$ 时的迎角是否为 0。
2. **位对齐校验**：对比 GPU (MC) 与 CPU (Debug) 的终点坐标，误差必须 $< 0.01\text{m}$。
3. **分阶段调优**：
   - 第一阶段：寻找无制导下的最优弹道仰角。
   - 第二阶段：在最优仰角基础上调优制导律参数（导航比、增益）。

### 3.2 目录规范
- **脚本**：`scripts/dart/`
- **数据**：`data/dart/`
- **日志**：`output/logs/`
- **图片**：`output/images/`

## 4. 常见陷阱 (Anti-Patterns)
- **禁止硬编码**：严禁在 `.cu` 或 `.cpp` 中硬编码质量、面积等参数，必须通过 `DartConfig` 统一管理。
- **禁止盲猜**：在命中率为 0% 时，必须优先检查受力正负号和坐标转换，而不是随机调参。
- **RK4 精度**：大气密度计算必须位于导数计算函数内部，确保积分阶段的物理一致性。
